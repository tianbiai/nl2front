> ⚠️ **必须遵循 `vue-development-guides` 核心原则**：
>
> - **状态可预测性**：单一数据源，其他状态由此派生
> - **明确的数据流**：Props down, Events up（大部分情况）
> - **组件小型化**：优先使用小型组件 + composables，而非大型组件
> - **避免不必要的重渲染**：明智使用计算属性和侦听器

**组件拆分策略**：

- [ ] 页面/元素已拆分为多个小型、专注的组件

- [ ] 复杂逻辑已提取到 composables（`useXxx()`）
- [ ] UI 部分已拆分为子组件（props in, events out）
- [ ] 状态/副作用已移至 composables
